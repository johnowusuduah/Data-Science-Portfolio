Comprehensive Regression Modeling Using Python (A Test Case to Predict House Prices in King County, Seattle, U.S.A)¶

Aim of Project¶

How does one use regression modeling with Python to make predictions when given data with numerous possible predictor variables? The goal of this notebook is to serve as a comprehensive guide on how to do this using data on houses in King County, Seattle, USA as a test case.

When I was learning regression modeling using Python, I wish I had a practical reference that synthetisized all the concepts and skils under the topic so that learning would have been easier. I have applied all the concepts under regression modeling on a test case to show how Python can be used to generate all types of regression models that can help other students and practitioners of data science to make meaninful predictions. In-sample model evaluation has been carried for demonstration purposes only to carry along anyone who might have difficulty understanding out-of-sample evaluation right of the bat. The sequencing, which is a key feature of this project, will be most beneficial to anyone who is struggling to synthesize and consolidate newly acquired skills and concepts in regression modelling.

Motivation¶

To make it easier for other students and practitioners of data science to learn and use regression modeling using Python.

Source of Data¶

Kaggle

Description of Data¶

This dataset contains house sale details for King County, which includes Seattle for the period May 2014 to May 2015.

id : A notation for a house; date: Date house was sold;price: Price is prediction target;bedrooms: Number of bedrooms;bathrooms: Number of bathrooms;sqft_living: Square footage of the home;sqft_lot: Square footage of the lot;floors :Total floors (levels) in house;waterfront :House which has a view to a waterfront;view: Has been viewed;condition :How good the condition is overall;grade: overall grade given to the housing unit, based on King County grading system;sqft_above : Square footage of house apart from basement;sqft_basement: Square footage of the basement;yr_built : Built Year;yr_renovated : Year when house was renovated;zipcode: Zip code;lat: Latitude coordinate;long: Longitude coordinate;sqft_living15 : Living room area in 2015(implies-- some renovations) This might or might not have affected the lotsize area;sqft_lot15 : LotSize area in 2015(implies-- some renovations)

Polynomial Regression with Multiple Independent Variables¶

Pipelines¶

What do we do when a linear model and a univariate polynomial model is not the best fit for our data? We evaluate our data using multivariate polynomial regression. This involves transforming our data into a polynomial and using linear regression to fit the data. However, this is convulated process for more than one independent variable hence, pipelines are employed to simplify our code and make it efficient.

Pipelines are employed to simplify long-winded code involving normalization, transforming and model generation using linear regression.

Structurally, pipelines consist of a list of tuples. The first element in the tuple contains the name of the estimator and the second element contains the the model constructor:

Estimator:

'scale'

'polynomial'

'model'

Model Constructor:

StandardScaler()

PolynomialFeatures(include_bias=False)

LinearRegression()

The default degree parameter of PolynomialFeatures is 2 so it is not necessary to include it but it has been included for demonstration purposes only.

In [158]:
Pipeline_Input=[('scale',StandardScaler()),('polynomial', PolynomialFeatures(degree=2)),('model',LinearRegression())]

I use the list - 'Input' to create a pipeline object to predict house prices, fit the object using the top five correlated variables (see feat_top5() above). We iterate through different orders of the polynomial function to determine which order will yield the highest coefficient of determination (R^2).

In [159]:
R_square_pipe = []
feat_top5 = ['sqft_living', 'grade', 'sqft_above', 'sqft_living15','bathrooms'] 
poly_order = range(2,11)

for n in poly_order:
    Input=[('scale',StandardScaler()),('polynomial', PolynomialFeatures(degree=n)),('model',LinearRegression())]
    pipe = Pipeline(Input)
    pipe.fit(df[feat_top5], df['price'])
    R_pipe = pipe.score(df[feat_top5], df['price'])
    R_square_pipe.append(R_pipe)

The greater the proportion of training data selected, the more accurate our prediction model will be. However, this comes at a cost of decreased precision of error between predicted values and actual values. To optimize accuracy and precision of our model, we cross-validate the data. Cross-validation is an effective way of using our data because every row or observation is used for training and testing.

One insight gained from performing in-sample evaluation was that we saw that a multivariate polynomial model fit our data best. Therefore we will first, cross-validate our data with our top 5 most correlated predictor values and iterate the order of the polynomial till we get a maximum R^2 value.

6: Model Refinement¶

Ridge Regression¶

The condition that justifies the use of ridge regression is when there is strong correlation among the predictor variables. This is known is multicollinearity. Regression models work best when changes to one independent variable does not affect other independent variables. When there is strong correlation between independent variables it makes it difficult for the model to estimate the relationship between each independent variable and the dependent variable, in an independent manner, because the independent variables change in unison. Muliticollinearity shows itself when coefficients vary wildy depending which combination of independent variables we choose.

Important Caveat

If the goal of regression modeling is only to make predictions, then it is imperative to note that multicollinearity does not negatively affect the accuracy of predictions. However, if the goal of carrying out regression modeling is to determine coefficients and p values then it is crucial that multicollinearity is dealt with.

"The fact that some or all predictor variables are correlated among themselves does not, in general, inhibit our ability to obtain a good fit nor does it tend to affect inferences about mean responses or predictions of new observations." — Applied Linear Statistical Models, p289, 4th Edition.

An overfitted model where the model tends to fit the noise is the result of multicollinearity. To avoid overfitting prevalent in high order polynomial models and models with a lot of features, ridge regression is performed. Ridge regression does this by controlling the magnitude of polynomial coefficents with the parameter "alpha".

We will use ridge regression to generate and fine tune our model with the train_test_split method. This is because the train_test_split method splits our data into train and test data so we can easily notice the improvements that ridge regression makes to our trained model when we juxtapose predicted target values to test values (which will be our validation data). **
